## Coverage Status Rules

- `contract`: the skill has per-skill contract coverage through `assert_skill_contract(...)` in its test module.
- `entrypoint`: the skill has explicit entrypoint ownership coverage through `assert_entrypoints_exist(...)`.
- `smoke spec`: every current script in that skill has a matching `tests/script_specs/...json` smoke spec.
- `partial smoke spec (N/M)`: only `N` of `M` current scripts have checked-in smoke specs.
- `tool availability`: the skill has no repo-local entrypoint scripts, but its tests assert external tool resolution.
